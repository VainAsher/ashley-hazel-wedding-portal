# Roadmap

## Phase 1 - Safe Prototype Baseline

Goal: make the current static prototype canonical, reviewable, and safe to iterate on.

- keep current visual concept intact
- define synthetic data rules
- add manual validation script
- avoid backend/storage until data boundary is approved

## Phase 1 Status

- WD-001 complete: static/browser-only interaction baseline validated and merged.
- WD-002 next: responsive and accessibility review readiness.

## Phase 2 - Usable Local Dashboard

Goal: make the dashboard useful with synthetic data.

- replace hard-coded repeated mock values with a small local fixture
- make RSVP and request flows interactive in browser state
- add clear empty/saved/error states
- improve mobile layout where needed
- keep all data local and synthetic

## Phase 3 - Real Data Readiness

Goal: prepare for real wedding data without exposing it.

- decide storage model
- define access model
- define backup/export rules
- define invite-code/household model
- run privacy review before real data entry

## Phase 4 - Stakeholder Validation

Goal: validate with Ashley/Hazel or trusted stakeholders using safe data.

- run manual validation script
- capture feedback
- prioritize final polish
